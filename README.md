# Интернет-магазин «Web-Larёk»

«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component

Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`

#### Класс Api

Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter

Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` - хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Модели данных

#### Класс Cart

Класс Cart представляет корзину пользователя и позволяет добавлять, удалять и управлять товарами в корзине.
Методы класса: - addItem(product: IProduct): void Добавляет товар product в корзину.
removeItem(product: IProduct): void Удаляет товар product из корзины.
getItemCount(): number Возвращает количество товаров в корзине.
getTotalCost(): number Возвращает общую стоимость товаров в корзине.
hasItem(id: string): boolean Проверяет наличие товара с указанным id в корзине и возвращает true или false.

#### Класс Customers

Класс Customers представляет данные покупателя и позволяет сохранять, получать и валидировать эти данные.
Методы класса:

- saveData(key: string, value: any): void Сохраняет данные value по ключу key.
  getAllData(): { [key: string]: any } Возвращает все сохранённые данные покупателя.
  validateData(): boolean Валидирует данные покупателя и возвращает true, если данные корректны, иначе false.
  resetData(): void Сбрасывает все данные покупателя.

#### Класс Catalog

Класс Catalog предназначен для работы с каталогом товаров. Он позволяет сохранять список товаров, получать список товаров, искать товар по идентификатору и управлять выбранным товаром.

Поля класса:

- products: IProduct[] — массив товаров. - selectedProduct: IProduct | null — выбранный товар или null, если товар не выбран.

Методы класса:
saveProducts(products: IProduct[]) Сохраняет массив товаров products в поле products.
getProducts(): IProduct[] Возвращает массив товаров, сохранённых в поле products.
getProductById(id: string): IProduct | undefined Ищет товар по идентификатору id в массиве products и возвращает его или undefined, если товар с указанным идентификатором не найден.

setSelectedProduct(product: IProduct) Устанавливает выбранный товар product в поле selectedProduct.
getSelectedProduct(): IProduct | null Возвращает выбранный товар из поля selectedProduct или null, если выбранный товар не установлен.

### Слой коммуникации

Класс ServerCommunication
Класс ServerCommunication предназначен для взаимодействия с сервером через API. Он использует объект api для отправки GET и POST запросов и получения данных.
Конструктор:
constructor(api: IApi)
Принимает объект api, который реализует методы для отправки запросов.
Методы класса:

- getProducts(): Promise<IProduct[]> Отправляет GET запрос на сервер для получения списка товаров и возвращает промис с массивом товаров типа IProduct.

placeOrder(data: any): Promise<IOrderResponse> Отправляет POST запрос на сервер для оформления заказа с данными data и возвращает промис с ответом сервера типа IOrderResponse.

### Слой Представления

#### Класс EmailPhoneForm

Назначение: форма для ввода email и номера телефона. Наследует базовую функциональность от класса Form.
Ключевые особенности: _ отслеживает изменения в полях «email» и «phone» в реальном времени; _ генерирует события при изменении значений полей и при отправке формы; _ позволяет программно устанавливать значения в полях; _ использует типобезопасность TypeScript (в т. ч. дженерики).
Интерфейс EmailPhoneFormData

interface EmailPhoneFormData {
email: string;
phone: string;
}

Описание: определяет структуру данных, которые форма будет обрабатывать.
Поля: _ email — строка с адресом электронной почты; _ phone — строка с номером телефона.
Зачем нужен: позволяет строго типизировать данные формы — это снижает риск ошибок при работе с ними.
Свойства класса
emailInputElement: HTMLInputElement
Тип: ссылка на DOM‑элемент (<input>).
Назначение: хранит элемент поля ввода email. Используется для:
получения текущего значения (value);
добавления обработчиков событий (addEventListener).
phoneInputElement: HTMLInputElement
Аналогично emailInputElement, но для поля телефона.
events: IEvents
Тип: экземпляр класса EventEmitter (реализующего интерфейс IEvents).
Назначение: механизм коммуникации формы с другими компонентами — через генерацию и подписку на события.
Конструктор constructor(container: HTMLElement, events: IEvents)
Что делает: инициализирует форму — находит поля ввода в DOM и настраивает обработчики событий.
Параметры: 1. container — HTML‑элемент‑контейнер, внутри которого размещена форма. 2. events — экземпляр EventEmitter для работы с событиями.
Ключевые шаги внутри конструктора:
Вызов super(container, events)
Передаёт аргументы базовому классу Form — инициализирует общую логику формы.
Поиск полей ввода
Использует вспомогательную функцию ensureElement, чтобы найти элементы по CSS‑селекторам:
"input[name=email]" → emailInputElement;
"input[name=phone]" → phoneInputElement.
Настройка обработчиков событий:
для поля email — слушает событие input и генерирует событие "email:input";
для поля телефона — аналогично, событие "phone:input";
для кнопки отправки (submitButton) — генерирует событие "contacts:submit" при клике.
Сеттеры (setters)
set email(value: string)
Что делает: устанавливает значение в поле email (this.emailInputElement.value = value).
Когда использовать: чтобы программно заполнить форму (например, данными из локального хранилища).
set phone(value: string)
Аналогично для поля телефона.
События формы
Форма генерирует следующие события через this.events.emit: _ "email:input" — когда пользователь меняет текст в поле email; _ "phone:input" — при изменении поля телефона; \* "contacts:submit" — при клике на кнопку отправки.
Как использовать: другие компоненты могут подписаться на эти события через events.on.

#### Класс ProductCard

interface ProductCardData {

Описание: определяет структуру данных, которые класс ProductCard будет обрабатывать.

titleElement: HTMLElement

priceElement: HTMLElement
Аналогично titleElement, но для цены продукта с классом .card__price.
Конструктор constructor(container: HTMLElement)
Что делает: инициализирует карточку продукта, находит элементы заголовка и цены в DOM.
Параметры: - container — HTML-элемент-контейнер, внутри которого размещена карточка продукта.
Ключевые шаги внутри конструктора:
Вызов super(container)
Передаёт аргументы базовому классу Component, инициализируя общую логику компонента.
Поиск элементов заголовка и цены
Использует вспомогательную функцию ensureElement, чтобы найти элементы по CSS-селекторам:
".card__title" → titleElement;
".card__price" → priceElement.
Сеттеры (setters)
set title(value: string)
Что делает: устанавливает текст заголовка продукта.
Пример использования: программно изменяет заголовок карточки.
set price(value: number | null)
Что делает: устанавливает текст элемента цены с учётом особого случая:

в противном случае — форматирует цену как строку с добавлением единицы измерения «синапсов» (например, 100 синапсов).

#### Класс ProductInBasket
Назначение: представляет карточку товара в корзине. Наследует функциональность от абстрактного класса ProductCard. Добавляет специфичную для корзины логику — отображение индекса товара и кнопку удаления.
Интерфейс ProductInBasketData

interface ProductInBasketData {
  index: number;
}
 
Описание: определяет дополнительные данные, специфичные для карточки товара в корзине.
Поле: - index — числовой индекс товара в корзине (например, порядковый номер).
Свойства класса
productIndexElement: HTMLElement
Тип: ссылка на DOM‑элемент, отображающий индекс товара.
Назначение: хранит элемент с классом .basket__item-index.
Используется для: отображения порядкового номера товара в корзине.
deleteButton: HTMLButtonElement
Тип: ссылка на кнопку удаления товара из корзины.
Назначение: хранит элемент кнопки с классом .basket__item-delete.
Используется для: привязки обработчика клика, который удаляет товар из корзины.
Конструктор constructor(container: HTMLElement, actions?: ICardActions)
Что делает: инициализирует карточку товара в корзине — находит дополнительные элементы (индекс и кнопку удаления) и настраивает обработчик события клика.
Параметры: 1. container — HTML‑элемент‑контейнер, внутри которого размещена карточка товара в корзине. 2. actions (опционально) — объект типа ICardActions, содержащий коллбэк onClick для обработки клика по кнопке удаления.
Ключевые шаги внутри конструктора:
Вызов super(container)
Передаёт контейнер базовому классу ProductCard — инициализирует общую логику карточки товара.
Поиск дополнительных элементов
Использует функцию ensureElement, чтобы найти:
".basket__item-index" → productIndexElement;
".basket__item-delete" → deleteButton.
Настройка обработчика клика
Если в actions присутствует метод onClick, привязывает его к кнопке удаления:

     this.deleteButton.addEventListener('click', actions.onClick);
 
Сеттер set index(value: number)
Что делает: устанавливает текстовое содержимое элемента, отображающего индекс товара в корзине.
Логика работы: 1. Принимает числовое значение value. 2. Преобразует его в строку с помощью String(value). 3. Устанавливает полученное значение в свойство textContent элемента productIndexElement.
Пример использования:

const productCard = new ProductInBasket(containerElement);
productCard.index = 3; // Отобразит «3» в элементе .basket__item-index
 
Особенности реализации
Наследование. Класс наследуется от ProductCard<ProductInBasketData>, что позволяет:
использовать всю базовую функциональность карточек товаров (заголовок, цена);
добавить специфичную для корзины логику (индекс, удаление).
Опциональные действия (actions). Использование опционального параметра actions делает класс гибким — можно:
не передавать обработчик, если кнопка удаления не нужна;
передать кастомный обработчик для выполнения нужной логики при удалении товара (например, запрос на сервер).
Строгая типизация. Благодаря дженерикам и интерфейсам:
данные строго типизированы;

#### Класс ProductInGallery
Назначение: представляет карточку товара в галерее. Наследует функциональность от абстрактного класса ProductCard. Добавляет логику отображения категории товара и его изображения.
Интерфейс ProductInGalleryData

interface ProductInGalleryData {
  category: string;
  image: string;
}
 
Описание: определяет дополнительные данные, специфичные для карточки товара в галерее.
Поля: * category — строка с названием категории товара. * image — строка, содержащая путь к изображению товара.
Свойства класса
categoryElement: HTMLElement
Тип: ссылка на DOM‑элемент, отображающий категорию товара.
Назначение: хранит элемент с классом .card__category.
Используется для: вывода названия категории и применения тематических стилей.
imageElement: HTMLImageElement
Тип: ссылка на элемент изображения товара.
Назначение: хранит тег <img> с классом .card__image.
Используется для: отображения картинки товара и установки атрибутов src и alt.
Конструктор constructor(container: HTMLElement, actions?: ICardActions)
Что делает: инициализирует карточку товара в галерее — находит элементы категории и изображения в DOM, настраивает обработчик клика по всей карточке.
Параметры: 1. container — HTML‑элемент‑контейнер, внутри которого размещена карточка товара. 2. actions (опционально) — объект типа ICardActions, содержащий коллбэк onClick для обработки клика по карточке.
Ключевые шаги внутри конструктора:
Вызов super(container)
Передаёт контейнер базовому классу ProductCard — инициализирует общую логику карточки товара.
Поиск дополнительных элементов
Использует функцию ensureElement, чтобы найти:
".card__category" → categoryElement;
".card__image" → imageElement.
Настройка обработчика клика
Если в actions присутствует метод onClick, привязывает его к контейнеру карточки:

    this.container.addEventListener('click', actions.onClick);
 
Сеттеры (setters)
set category(value: string)
Что делает: устанавливает название категории и добавляет тематический класс к элементу categoryElement.
Логика работы:
устанавливает textContent элемента categoryElement равным value;
ищет соответствующий класс в categoryMap и добавляет его к categoryElement.classList.
set image(value: string)
Что делает: задаёт путь к изображению товара и устанавливает атрибут alt.
Пример:

       this.imageElement.src = CDN_URL + value;
       this.imageElement.alt = this.titleElement.textContent || 'Товар';
 
Метод render(data: any): HTMLElement
Что делает: принимает данные и заполняет поля карточки товара, возвращая контейнер HTMLElement.
Параметры: * data — объект с данными для отображения (title, price, category, image).
Пример использования:

const productCard = new ProductInGallery(containerElement);
productCard.render({
  title: "Название товара",
  price: 100,
  category: "Электроника",
  image: "/images/product.jpg"
});
 
Логика работы: 1. Устанавливает заголовок (this.title = data.title). 2. Устанавливает цену (this.price = data.price). 3. Устанавливает категорию (this.category = data.category). 4. Устанавливает изображение (this.image = data.image). 5. Возвращает контейнер карточки (this.container).
Особенности реализации
Наследование. Класс наследуется от ProductCard<ProductInGalleryData>, что позволяет:
использовать всю базовую функциональность карточек товаров (заголовок, цена);
добавить специфичную для галереи логику (категория, изображение).
Динамическая стилизация. Метод set category использует categoryMap для автоматического добавления CSS‑классов — это позволяет визуально различать товары по категориям.
Использование CDN. При установке изображения используется константа CDN_URL — это гарантирует корректную загрузку картинок с CDN.
Гибкость обработчиков. Опциональный параметр actions позволяет настраивать поведение карточки (например, переход на страницу товара при клике).
Метод render. Централизует логику заполнения карточки данными — упрощает повторное использование компонента.

#### Класс ProductPreview
Назначение: представляет карточку товара в режиме предпросмотра. Наследует базовую логику от абстрактного класса ProductCard и добавляет функциональность для отображения категории, описания, изображения и кнопки с возможностью управления её состоянием.
Интерфейс ProductPreviewData

interface ProductPreviewData {
  category: string;
  description: string;
  image: string;
  buttonText: string;
}
 
Описание: задаёт структуру данных, необходимых для заполнения карточки предпросмотра товара.
Поля: - category — название категории товара (строка); - description — текстовое описание товара (строка); - image — путь к изображению товара (строка); - buttonText — текст, который отображается на кнопке карточки (строка).
Свойства класса
categoryElement: HTMLElement
Элемент DOM, отображающий категорию товара. Имеет класс .card__category.
imageElement: HTMLImageElement
HTML‑элемент <img>, предназначенный для отображения изображения товара. Класс — .card__image.
descriptionElement: HTMLElement
Элемент, в котором отображается описание товара. Класс — .card__text.
cardButton: HTMLButtonElement
Кнопка, связанная с карточкой товара. Класс — .card__button. При клике генерирует событие product:choose.
events: IEvents
Объект, реализующий интерфейс IEvents. Используется для работы с событиями, в т. ч. для эмита события при нажатии кнопки.
Конструктор constructor(container: HTMLElement, protected events: IEvents)
Параметры:
container — HTML‑элемент, который служит контейнером для карточки товара;
events — объект типа IEvents, используемый для работы с событиями.
Что делает:
Вызывает конструктор базового класса ProductCard.
Находит необходимые DOM‑элементы с помощью ensureElement.
Устанавливает обработчик клика на cardButton, который при нажатии генерирует событие product:choose.
Сеттеры (setters)
set category(value: string)
Устанавливает название категории и добавляет соответствующий CSS‑класс из categoryMap.
set image(value: string)
Задаёт путь к изображению товара, используя CDN_URL.
set description(value: string)
Заполняет описание товара.
set buttonText(value: string)
Обновляет текст на кнопке карточки.
Метод buttonProhibited(value: boolean)
Цель: включает или отключает кнопку на карточке товара в зависимости от переданного значения.
Пример использования:

this.buttonProhibited(true); // кнопка будет заблокирована

#### Класс BasketModal

Назначение: представляет модальное окно корзины покупок. Предназначен для отображения списка товаров, общей цены и кнопки для оформления заказа.
Интерфейс BasketModalData

interface BasketModalData {
  items: HTMLElement[];
  totalPrice: number;
}
 
Описание: определяет структуру данных, необходимых для работы модального окна корзины.
Поля: * items — массив элементов DOM, представляющих товары в корзине; * totalPrice — общая цена всех товаров в корзине.
Свойства класса
listElement: HTMLElement
Тип: ссылка на DOM‑элемент, отображающий список товаров в корзине.
Назначение: хранит элемент с классом .basket__list.
registerButton: HTMLButtonElement
Тип: ссылка на кнопку оформления заказа.
Назначение: хранит элемент кнопки с классом .basket__button.
Используется для: генерации события basket:submit при клике.
totalPriceElement: HTMLElement
Тип: ссылка на элемент, отображающий общую цену.
Назначение: хранит элемент с классом .basket__price.
Конструктор constructor(container: HTMLElement, protected events: IEvents)
Что делает: инициализирует модальное окно корзины — находит необходимые элементы в DOM и настраивает обработчик клика для кнопки оформления заказа.
Параметры: 1. container — HTML‑элемент‑контейнер, внутри которого размещено модальное окно. 2. events — объект типа IEvents, используемый для работы с событиями.
Ключевые шаги внутри конструктора:
Вызов super(container)
Передаёт контейнер базовому классу Component.

Использует функцию ensureElement, чтобы найти:

Что делает: устанавливает общую цену в корзину.

set registerButtonDisabled(value: boolean)
Что делает: включает или отключает кнопку оформления заказа.
Логика работы:

      this.registerButton.disabled = value;

#### Класс ModalWindow

Назначение: реализует модальное окно — всплывающий блок, который перекрывает основной контент страницы. Позволяет отображать произвольное содержимое, открывать и закрывать окно по клику.
Интерфейс ModalData

interface ModalData {
  content: HTMLElement;
}
 
Описание: задаёт структуру данных, необходимых для работы модального окна.
Поля: - content — HTML‑элемент, который будет отображаться внутри модального окна (его содержимое).
Свойства класса
closeButton: HTMLButtonElement
Тип: ссылка на DOM‑элемент кнопки закрытия окна.
Назначение: хранит элемент с классом .modal__close.
Используется для: генерации события modal:close при клике на кнопку.
modalContainer: HTMLElement
Тип: ссылка на контейнер, в котором размещается содержимое окна.
Назначение: хранит элемент с классом .modal__content.
Используется для: вставки и замены содержимого модального окна через replaceChildren.
Конструктор constructor(container: HTMLElement, protected events: IEvents)
Цель: инициализировать модальное окно, найти ключевые элементы в DOM и настроить обработчики событий.
Параметры: * container — корневой HTML‑элемент модального окна;
* events — объект типа IEvents, необходимый для работы с событиями.
Что делает: 1. Вызывает конструктор базового класса Component (super(container)).
2. С помощью ensureElement находит: * кнопку закрытия (.modal__close) → closeButton; * контейнер для содержимого (.modal__content) → modalContainer.
3. Настраивает два обработчика: * клик на closeButton → отправляет событие modal:close; * клик по всему контейнеру (this.container) → закрывает окно, если клик пришёлся на фон.
Сеттер set content(item: HTMLElement)
Цель: установить содержимое модального окна и сразу его открыть.
Параметр:
item — HTML‑элемент, который станет содержимым окна.
Что делает: 1. Очищает текущее содержимое modalContainer и вставляет новый элемент (replaceChildren).
2. Вызывает метод open() для показа окна.
Методы
open(): void
Цель: показать модальное окно.
Как работает: добавляет CSS‑класс modal_active к container.
Эффект: визуально отображает окно (например, делает его видимым через переходы/анимации).
close(): void
Цель: скрыть модальное окно.
Как работает: удаляет CSS‑класс modal_active из container.
Эффект: скрывает окно (например, убирает его из вида или возвращает в скрытое состояние).

#### Класс Form
Назначение: предоставляет базовую функциональность для работы с веб‑формами. Содержит общие методы и свойства, которые можно использовать в конкретных реализациях форм.
Интерфейс FormData

interface FormData {
  errors: string;
}
 
Описание: задаёт структуру данных, общих для всех форм.
Поля: * errors — строка с текстом ошибок валидации формы.
Свойства класса
submitButton: HTMLButtonElement
Тип: ссылка на DOM‑элемент кнопки отправки формы.
Назначение: хранит кнопку с атрибутом type="submit".
Используется для: управления состоянием кнопки и предотвращения стандартной отправки формы.
errorsElement: HTMLElement
Тип: ссылка на элемент, отображающий ошибки формы.
Назначение: хранит элемент с классом .form__errors.
Используется для: вывода текста об ошибках валидации.
Конструктор constructor(container: HTMLElement, protected events: IEvents)
Цель: инициализировать базовую логику формы — найти ключевые элементы в DOM и настроить обработчик отправки.
Параметры: * container — корневой HTML‑элемент формы; * events — объект типа IEvents, используемый для работы с событиями.
Что делает: 1. Вызывает конструктор базового класса Component (super(container)). 2. С помощью ensureElement находит: * кнопку отправки (button[type=submit]) → submitButton; * элемент для отображения ошибок (.form__errors) → errorsElement. 3. Настраивает обработчик клика на submitButton, который: * предотвращает стандартную отправку формы (e.preventDefault()).
Методы
isallowedButton(value: boolean)
Цель: включить или отключить кнопку отправки формы.
Параметр: value — булево значение:
true — кнопка будет отключена (disabled=true);
false — кнопка будет активна (disabled=false).
Что делает: изменяет атрибут disabled у submitButton.
set errors(item: string)
Цель: установить текст об ошибках и отобразить его в интерфейсе.
Параметр: item — строка с текстом ошибок.
Что делает: задаёт textContent элемента errorsElement равным переданной строке.
Особенности реализации
Дженерик <T>. Класс использует дженерик, чтобы быть универсальным: конкретная реализация формы может добавить свои данные (например, поля ввода) через тип T.
Абстрактный класс. Поскольку класс помечен как abstract, его нельзя использовать напрямую — только как основу для конкретных форм (например, LoginForm, RegistrationForm и т. п.).
